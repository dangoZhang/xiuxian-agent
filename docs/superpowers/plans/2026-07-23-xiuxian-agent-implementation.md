# 修仙agent：实施计划

日期：2026-07-23
依据：[设计规格](../specs/2026-07-23-xiuxian-agent-design.md)

## 目标

交付一个必须连接 OpenAI-compatible 大模型 API 才能开局的完整 Web 游戏。玩家用自然语言控制主角；天道生成规则，命运生成事件，六名修士 Agent 在神识与全局并发限制内调用工具、交谈和战斗；全部内容沿时间轴显示并可存档。

## 视觉 brief

- 概念：夜色中的“活墨卷”，时间轴像命线，模型调用像神念沿线游走。
- 色彩：玄黑、宣纸白、暗金；朱砂用于战斗，青玉用于玩家，幽紫用于命运。
- 字体：中文衬线正文，窄体无衬线用于数据与状态。
- 密度：正文宽松，状态栏紧凑。
- 表面：细纹纸张、印章、双线与断线边框；不使用玻璃卡片堆叠。
- 动效：事件淡入、神念流动、战斗断线闪烁；支持 reduced motion。

## 工作包

### 1. 工作区

创建 pnpm workspace、TypeScript 基础配置、统一脚本、环境示例、MIT License、GitHub Actions 与 Docker 配置。

验收：`pnpm typecheck`、`pnpm test`、`pnpm build` 可从根目录运行。

### 2. 共享协议

在 `packages/protocol` 定义并导出：

- 模型配置与能力校验 schema。
- 天道 DSL、世界、角色、功法、工具调用、命运事件 schema。
- 时间轴记录、队列事件、游戏状态、SSE 事件与导入导出 schema。
- 所有 source、kind、phase 和错误代码。

验收：非法 source、超预算工具、损坏存档和不支持的 DSL 无法通过解析。

### 3. 引擎

在 `packages/engine` 实现：

- 稳定优先队列与最多 32 微步的按需推进。
- 天道 DSL 编译与领域事件归约。
- 神识账本、加权信号量、Agent 唤醒筛选。
- 工具校验和行动草稿事务。
- 顺序无关的战斗结算与状态哈希。
- 存档投影与恢复。

验收：队列跳日、并发上限、工具成本、战斗换序、状态哈希均有单元测试。

### 4. 提示词与模型层

在 `packages/prompts` 与 `apps/server` 实现：

- OpenAI-compatible provider 工厂。
- 结构化输出、工具调用、流式文本三项真实能力校验。
- 天道、命运、玩家意图、修士回合、战斗叙事提示词。
- 每个修士只收到可见状态和有限记忆。
- 单次结构化输出修复；失败后暂停原 revision。

验收：无 API 或任一能力不满足时拒绝创建会话；生产代码不包含模拟内容。

### 5. 游戏服务

在 `apps/server` 实现：

- Hono REST API 与 SSE 事件流。
- API Key 仅存于会话内存。
- SQLite 游戏、事件和存档仓库。
- 开局编排、玩家命令、相关 Agent 并行行动、战斗、推进、导出与导入。
- 事务失败回滚，已结算事件原子写入。

验收：服务接口与设计规格一致；并发事件均带 `callId/source/actorId/sequence`。

### 6. Web 游戏

在 `apps/web` 实现：

- 模型设置与身世开局页。
- 桌面三栏与移动端正文/时间轴/角色切换。
- 四权来源组件、战斗视觉、因果追溯。
- 神识、并发调用、角色关系、功法与资源状态。
- SSE 消费、玩家输入、推进、重试、导入导出。

验收：390×844 与 1440×900 无横向溢出；键盘焦点、对比度和 reduced motion 合格。

### 7. 开源交付

补齐 README、架构图、配置说明、开发指南、贡献指南、License、Docker Compose 和 LiteLLM 子模块说明。初始化并固定 `vendor/litellm`。

验收：新用户按 README 可完成安装、配置 API、启动和开局。

### 8. 完成审计

逐项核对设计规格第 13、14 节。执行类型检查、单元测试、构建、浏览器交互与响应式截图。修复后提交，并发布到用户 GitHub 仓库 `修仙agent`。
